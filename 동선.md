## 1. 워커 동선 (이상 흐름)

```
로그인
  └→ 시프트 [탭1]
       ├→ 카드 탭 → /cafe/[id] (매장 신뢰도 + 시프트 목록)
       │    └→ "1탭 지원" → 본인 PENDING 상태 표시
       └→ 매칭 확정 알림 종
            └→ 내 매칭 [탭2]
                 ├→ 매장 탭 → /cafe/[id]
                 ├→ "체크인" → IN_PROGRESS
                 ├→ "체크아웃" → CHECKED_OUT
                 │    └→ (제안) 정산 요청 자동 진입 → REQUESTED 상태 라벨
                 ├→ "매장 평가하기" → RatingModal (worker-rates-owner)
                 └→ "채팅" → ChatModal
            
       └→ 점주 승인 후
            └→ 내 정산 [탭3]
                 ├→ 정산 카드 → 매장명 + 매장진입 + 계약서/영수증 4 quick action
                 ├→ /contract/[matchId]
                 └→ /withholding/[matchId]
       
       └→ 본인 통계는 [탭4]
            ├→ 받은 평가 → 매장진입 + 별점 분포 5/4/3/2/1 막대
            └→ 카드 탭 → /cafe/[id]
```

### 워커가 받는 알림 → 가야 할 페이지

| 타입 | 트리거 | 현재 라우트 | **이상 라우트** | 현재 상태 |
|---|---|---|---|---|
| `APPLICATION_REJECTED` | 점주가 다른 워커 accept | `/cafe/{id}` | `/cafe/{id}` (그대로 OK — 다른 매장 시프트 보러 갈 동기) | ✅ |
| `APPLICATION_AUTO_WITHDRAWN` | 1h 버퍼 충돌 자동 철회 | `/cafe/{id}` | `/worker/matches` (매칭된 시프트 확인이 더 자연스러움) | ⚠️ 변경 필요 |
| `NEW_MATCH` | 점주 accept | `/worker/matches` | `/worker/matches?focus={matchId}` (포커스 스크롤) | ⚠️ 미구현 |
| `NEEDS_RATING` | 본인 체크아웃 후 평가 미완 | `/worker/matches` | `/worker/matches?focus={matchId}&action=rate` (모달 자동 오픈) | ⚠️ 미구현 |
| `OWNER_RATING` | 점주가 나에게 평가 남김 | `/worker/stats` | `/worker/stats` (그대로 OK) | ✅ |
| `PAYOUT_COMPLETED` | 입금 완료 | `/worker/payouts` | `/worker/payouts?focus={payoutId}` 또는 `/contract/{matchId}` | ⚠️ 변경 필요 |

## 3. 대시보드 카드 → 상세 동선 (제안)

현재 상태: **카드는 표시만, 클릭 안 됨.** (`shifts.tsx:249~252`)

### 제안: 카드 = 필터 단축 버튼

```
[모집중 N개]   ← 클릭 → setFilter('OPEN')  + 칩에 active 표시 + 스크롤 to 목록
[매칭완료 N개] ← 클릭 → setFilter('MATCHED') 
[진행중 N개]   ← 클릭 → setFilter('IN_PROGRESS') (현재 'MATCHED'에 흡수됨, 분리 필요)
[완료 N개]     ← 클릭 → setFilter('COMPLETED')
```

- 즉 별도 화면 만들 필요 없음 — 같은 화면에서 필터 칩과 동기화.
- 추가 작업: `FilterChip` 상태에 `'IN_PROGRESS'` 분리 (현재 'MATCHED'에 IN_PROGRESS 포함).
- StatCard 컴포넌트에 `onPress` prop 추가, ScrollView ref로 목록 위치로 스크롤.

### 더 깊이 가고 싶다면 (Phase 2)

매장별 / 기간별 매트릭스 화면을 새 라우트로:
```
/owner/dashboard/[status]    # 예: /owner/dashboard/open
   ├→ 기간 필터 (오늘/7일/30일)
   ├→ 매장별 분포 차트
   └→ 시프트 목록 (드래그 정렬, 일괄 액션)
```

## 4. 핵심 워크플로우 (지원 → 정산까지)

### 4-1. 현재 흐름

```
워커: 시프트 검색 → 1탭 지원
                       ↓ [지원 PENDING]
점주: 시프트 상세 → 워커 accept
                       ↓ [match MATCHED, 다른 PENDING REJECTED, 1h 충돌 PENDING WITHDRAWN]
                       ↓ [shift MATCHED]
워커: 시프트 시작 → 체크인
                       ↓ [match CHECKED_IN, shift IN_PROGRESS]
                  [노쇼: NoShowScheduler 시작+30분 무체크인 → NO_SHOW + 자동 백업]
                       ↓
워커: 근무 종료 → 체크아웃
                       ↓ [match CHECKED_OUT, shift COMPLETED]
                       ↓ [Payout SCHEDULED 자동 생성]
                       ↓ [PayoutScheduler 즉시 mock COMPLETED — 30분 SLA 자동 충족]
                       ↓
양방향 평가 (자유 시점, 강제 X)
   - 워커 → 매장 (RatingModal worker-rates-owner)
   - 점주 → 워커 (RatingModal owner-rates-worker)
```

### 4-2. 제안 흐름 (앞 라운드 미구현 항목)

```
워커: 1탭 지원
점주: accept
워커: 체크인
워커: 체크아웃
                       ↓ [Payout REQUESTED — 점주 승인 대기로 변경]
점주: 시프트 상세 "정산 승인 + 평가" 한번에
   - RatingModal (별점/재고용/코멘트) 필수
   - 승인 클릭 → Rating 저장 + Payout SCHEDULED → 30분 SLA 시작
                       ↓
PayoutScheduler → COMPLETED → 워커 알림(PAYOUT_COMPLETED)
                       ↓
워커: 매장 평가 (NEEDS_RATING 알림 → /worker/matches?action=rate)
                       ↓
양쪽 모두 평가 완료 = 매칭 사이클 종료
```

### 4-3. 노쇼 분기 (제안)

```
[자동] NoShowScheduler — 시작+30분 무체크인 → NO_SHOW + 백업 매칭 (현존)

[수동] 점주: 시프트 상세 "노쇼 등록" 버튼 (matched 단계에서만 노출)
   ├→ POST /api/owner/matches/{matchId}/no-show
   ├→ match.NO_SHOW
   ├→ 자동 Rating(OWNER_TO_WORKER, ★1, willRehire=false, comment="워커 미출근")
   ├→ 백업 매칭 시도 (NoShowService 재사용)
   └→ 워커 알림 발송 (NOSHOW_REPORTED 새 타입)
```

## 5. 갭 분석 (현재 ↔ 이상)

| 영역 | 현재 | 갭 | 우선순위 |
|---|---|---|---|
| 알림 → 특정 상세 진입 | 일부는 일반 목록으로만 이동 | `?focus={id}&action=rate` 패턴으로 모달 자동 오픈 | 🔥 P0 |
| 대시보드 카드 클릭 | 표시만, 클릭 X | 같은 화면 필터 칩과 동기화 | 🔥 P0 |
| 진행중/매칭완료 분리 | FilterChip 'MATCHED' 하나로 흡수 | 분리해서 4개 카드 = 4개 필터 1:1 | 🔥 P0 |
| 정산 승인 절차 | 자동 (점주 승인 X) | Payout REQUESTED → 점주 승인 → SCHEDULED | 🟡 P1 |
| 점주 평가 강제 | 자유 시점 | 정산 승인 모달에 통합, 평가 없이 승인 불가 | 🟡 P1 |
| 노쇼 수동 등록 | 자동만, 점주 액션 없음 | 시프트 상세에 "노쇼 등록" 버튼 + 자동 ★1 평가 | 🟡 P1 |
| 워커 정산 상태 라벨 | 입금완료만 보임 | 정산요청/승인대기/입금예정/입금완료 4단계 | 🟢 P2 |
| 매장 평가 알림 라우팅 | `/owner/shifts` 일반 목록 | `/cafe/{cafeId}?tab=reviews` | 🟢 P2 |
| Phase 2 대시보드 별도 화면 | 없음 | `/owner/dashboard/[status]` 매트릭스 | 🟢 P2 |
