## 0. Executive Summary (1쪽 요약)

**단바**는 한국 카페·베이커리 점주가 **1시간 안에** 검증된 알바를 매칭받고, 워커는 근무 종료 **30분 안에** 일급을 입금받는 **스팟워크 매칭 플랫폼**입니다.

- **시장**: 한국 카페·베이커리 약 9.6만 개 (커피전문점 9만+ / 베이커리 약 6천), 전체 단기알바 시장의 핵심 진입점
- **벤치마크**: 일본 Timee — 누적 가입자 1,500만+, 등록 사업장 28만+, 2024년 도쿄증권거래소 상장(시총 약 1조 엔)
- **차별점**: GPS 100m 출근 게이트, 노쇼 자동 백업 매칭, 양방향 평가, 단골 1탭 재고용, 30분 자동 정산 SLA
- **수익모델**: 점주 매칭 수수료 12% + 일용근로 원천징수 6.6% 자동 처리
- **현황**: MVP 풀스택 완성, 13개 핵심 기능 가동, 수직계열화(직무 7종 × 등급 4단계 × 자격 7종) 완료
- **자금 요청 (placeholder)**: Seed Round ₩{TBD}억 — 12개월 GTM 및 운영팀 구축

**원라이너**: *"1시간 매칭, 30분 입금, 노쇼 0% — 알바 펑크가 매장을 멈추지 않게 한다."*

## 1. 문제 정의 (Problem)

### 1.1 점주 측 — "사람이 안 와서 셔터 내린다"

- 카페·베이커리는 **피크타임 1~2명 펑크 = 매장 마비**. 신촌·홍대 상권 피크 매출의 30~40%가 4시간 안에 발생
- 기존 채널(알바몬·당근알바·인스타 DM)은 **이력서 기반·1주 단위 채용** — 당일 펑크에 무력
- 펑크 발생 시 점주가 직접 매장 일하면서 후임 워커를 폰으로 찾는 악순환

### 1.2 워커 측 — "용돈 일거리는 너무 번거롭다"

- 대학생·N잡러는 **빈 시간 4~6시간만 일하고 싶다** — 그러나 알바 등록 → 면접 → 합격까지 1~2주
- 일하고 나서도 **급여는 다음달 25일** — 스팟성 노동에 맞지 않는 정산 사이클
- 채용 후 **갑작스러운 출근 강요·휴게시간 미보장** 등 점주 신뢰도 정보가 0

### 1.3 시장 데이터 (placeholder — 출처 보강 필요)

- 한국 커피전문점 사업체: **약 9.6만 개** (통계청, 2024)
- 일평균 카페 알바 수요: **추정 5~7만 시프트/일**
- 일본 Timee MAU: **약 800만 명** (2024 결산자료) → 한국 인구 대비 환산 시 **MAU 200만~300만 잠재**

## 2. 솔루션 (Solution) — 단바

### 2.1 3대 SLA (정량 약속)

| 약속 | 정의 | 백업 메커니즘 |
|------|------|---------------|
| **1시간 매칭** | 점주 시프트 등록 후 60분 안에 워커 매칭 | 추천 알고리즘 + 단골 우선 알림 + 1탭 지원 |
| **30분 입금** | 워커 체크아웃 후 30분 안에 일급 송금 | 점주 즉시 승인 + 30분 자동 승인 안전망 |
| **노쇼 0%** | 시프트 시작 +30분 미체크인 → 자동 백업 | NoShowScheduler 매분 cron + 백업 후보 자동 ACCEPT |

### 2.2 핵심 차별점 (경쟁사 대비)

1. **GPS 100m 출근 게이트** — 매장 반경 안에서만 체크인 가능 (부정 출근 차단)
2. **양방향 평가** — 점주↔워커 모두 평가, 매장 신뢰도(★+노쇼율) 워커에게 노출
3. **단골 1탭 재고용** — 점주가 즐겨찾기 워커에게 시프트 직접 초대, 워커는 1탭 수락
4. **수직계열화** — 직무 7종(바리스타/홀/캐셔/베이커/주방/오픈/마감) × 등급 4단계(L1~L4) × 자격 7종(보건증/야간/POS/라떼아트 등) 매칭
5. **근로계약서 양측 ack 강제 게이트** — 워커 미확인 시 출근 차단, 점주 미확인 시 정산 차단 (분쟁 사전 차단)
6. **자동 원천징수** — 일급 15만원↑ 시 6.6% 자동 처리 (점주 세무 부담 0)

### 2.3 사용자 동선

**점주**: 매장 등록(카카오 Local API 자동 입력) → 시프트 등록(직무·등급·자격 지정) → 1시간 안에 매칭 → 워커 출근 → 정산 승인 + 평가 → 30분 안에 송금 완료

**워커**: 카카오 로그인 → 능력 자기신고(L1~L4) → 추천순 시프트 1탭 지원 → 매칭 → 근로계약서 ack → GPS 체크인 → 근무 → 체크아웃 → 30분 안에 입금

## 3. 시장 (Market)

### 3.1 TAM / SAM / SOM (placeholder)

- **TAM**: 한국 단기알바 시장 — 연간 거래액 약 ₩{TBD}조 (전체 알바몬·당근알바·자비스 합산)
- **SAM**: 카페·베이커리·디저트샵 스팟워크 — 일 5만 시프트 × 시급 ₩11,000 × 5h × 365일 ≈ **연 ₩1조**
- **SOM (3년 목표)**: SAM의 5% 점유 — **연 거래액 ₩500억, 수수료 매출 ₩60억**

### 3.2 일본 Timee 검증 사례

- 2018 창업 → 2024 도쿄증권거래소 상장 (시총 약 ₩1조)
- 매출 성장: 2022 41억엔 → 2023 144억엔 → 2024 268억엔 (3년 6.5x)
- **한국 시장이 4~5년 lag 인정 시 Timee 2020년 시점 도달 = 연 매출 ₩500억대**

### 3.3 카페→인접 업종 확장 (Phase 2~4)

> 핵심 메시지: **현재 도메인의 80%는 그대로 재사용 가능** — `Cafe→Workplace` 일반화 수준의 가벼운 추상화로 인접 업종을 단계 진입할 수 있음. 따라서 SAM 은 정적이지 않고 **확장 가능한 자산**.

#### 3.3.1 업종별 적합도 매트릭스 (SAM 동반)

평가 기준: **(긴급성) × (시급단가) × (규제 부담↓) × (1회성 적합도) × (현 도메인 재사용도)**

| 업종 | 적합도 | 핵심 이유 | 예상 SAM (연) | 진입 Phase |
|------|--------|-----------|--------------|-----------|
| **카페·커피전문점** | ★★★★★ | 현재 MVP | ₩1조 | Phase 1 (현재) |
| **베이커리·디저트샵** | ★★★★★ | 카탈로그 보유, 코드 변경 0 | ₩2,000억 | Phase 1 (현재) |
| **펍·이자카야·와인바** | ★★★★★ | 저녁 피크 + 시급↑ + 펑크 빈발 → "노쇼 0%" 가치 max | ₩5,000억 | Phase 2 |
| **음식점 홀·주방** | ★★★★★ | JobRole 추가만으로 진입. 시장 카페의 **5~10배** | ₩5~10조 | Phase 2 |
| **편의점·무인매장** | ★★★★☆ | 야간 펑크 빈발, GPS+CCTV로 부정 적음 | ₩8,000억 | Phase 3 |
| **행사·팝업·전시 스태프** | ★★★★☆ | 본질이 1회성. "주말 페어 안내요원 50명" bulk 매칭 | ₩3,000억 | Phase 3 |
| **돌봄·방과후 (Skima Care)** | ★★★★☆ | 단가·임팩트 최고. 검증 인프라 추가 필요 | ₩1조+ | Phase 4 (조건부) |

부적합: 미용실(단골 의존), 물류(쿠팡플렉스 dominant), 청소(청소연구소 점유), 의료·약국(면허 검증 부담), 배달(레드오션) — 단바 모델과 결이 어긋남.

#### 3.3.2 SAM 누적 시나리오

```
Phase 1 (현재):       카페·베이커리                     →  SAM ₩1.2조
Phase 2 (12개월 내):  + 펍·이자카야 + 음식점 일부       →  SAM ₩2~3조
Phase 3 (24~36개월):  + 편의점 + 행사·팝업              →  SAM ₩4~5조
Phase 4 (장기, 조건부): + 돌봄·방과후 (Skima Care 분기)  →  SAM ₩5~6조
```

**투자 함의**: SOM(3년 ₩500억) 달성 시점에 SAM 자체가 **2~3배로 확장** → Series A·B 라운드의 시장 사이즈 메시지가 현 SAM에 묶이지 않음.

#### 3.3.3 진입 옵션 3가지

| 옵션 | 작업량 | 도메인 재사용 | 임팩트 | 결정 |
|------|--------|--------------|--------|------|
| **A. 음식점·펍 수평 확장** | 1~2일 | 99% | 시장 5~10배 즉시 확보, PMF 신호 가속 | **✅ 진행 (Phase 2)** |
| **B. Skima Care vertical (보육·유치원 긴급 대체)** | 2~3주 + 비-코딩 | 80% | 단가·임팩트 최고. 사고 리스크·보험·자격검증 부담 | ⏸ 보류 (조건부 Phase 4) |
| **C. 멀티 vertical 플랫폼화** | 4~6주 | 추상화 신설 | 카탈로그만 추가하면 어떤 업종이든 운영 | ⏸ B 검증 후 재평가 (premature 위험) |

**A 우선 segment**: **펍·이자카야·와인바부터** — 저녁 피크·시급↑·펑크 빈발 조합으로 단바의 "노쇼 0%·30분 입금" 가치가 카페보다 더 강하게 팔림.

#### 3.3.4 Skima Care(B) 진입 조건 — Phase 4 트리거

B 트랙은 **현재 보류**. 진입 트리거:
1. **PL 보험 상품 확보** — 영유아 사고 리스크 분리
2. **보육·유치원 영업 채널 확보** — 현재 0건 (사용자 보유는 방과후만)
3. **자격증 검증 인프라** — `CredentialVerification` 도메인 + 보건복지부 자격관리시스템 연동 또는 점주 수동 검증
4. **신원조회 흐름** — 아동복지법 §29의3 의무 이행 (성범죄·아동학대 경력)

**진짜 wedge**: "보육·유치원 9시 등원 전 1시간 대체교사" — 단바 정체성(1시간 매칭·노쇼 0%)이 카페보다 훨씬 절박하게 팔리는 시장. 시급 카페의 **2.5~3배** (₩2.5~4만), 거래당 수수료 매출 **3~4배**.

**보류 이유 (현시점)**: 사고 리스크 = 회사 망함. 검증·보험·영업 비-코딩 부담 큼. 일정 예측 불가.

#### 3.3.5 도메인 변경 비용 (참고)

- **그대로 쓰는 것 (80%)**: `Shift`, `ShiftMatch`, `Payout`, `Rating`, GPS 체크인, 채팅, 노쇼 백업, 알림
- **일반화 필요**: `Cafe` → `Workplace`, `JobRole` 업종별 트리, `CafeType` 확장 (`Requirements`는 이미 자유형)
- **신규 필요 (B 트랙만)**: `CredentialVerification`, `BackgroundCheckReport`

→ 새 프로젝트가 아니라 **"Skima Care" 같은 vertical 한 줄 추가**로 갈 수 있음. 기술 부채는 작고, 시장 기회는 큼.

## 4. 제품 (Product)

### 4.1 현재 완성된 13개 기능 (Phase 1 + Phase 2 일부)

1. 시프트 등록(단일/일괄·요일패턴) + 매장 등록(카카오 Local API 자동 입력)
2. 1탭 지원 + 추천순 정렬(TrustScore 25% + 별점 20% + 노쇼율 10% + 거리 15% 등)
3. GPS 출근 인증(반경 100m) + 노쇼 30분 grace + 자동 백업 매칭
4. 양방향 평가 + 매장/워커 TrustScore (별점·재고용·노쇼·정산속도 가중)
5. 30분 자동 정산 안전망 + 일용근로 원천징수 6.6% 자동
6. 근로계약서 양측 ack 강제 게이트 + 원천징수영수증 발급
7. 수직계열화 매칭(직무 7종 × 등급 4단계 × 자격 7종)
8. 단골 1탭 재고용(시프트 직접 초대) + 즐겨찾기 양방향
9. 시프트 템플릿 + 자동 반복 등록(매주 X요일)
10. 분쟁 이의제기(24h 내, 자동 판정 휴리스틱)
11. 카카오 로그인 + 카카오맵 매장 지도 노출
12. 알림 인박스(read/unread, 30s 폴링) + 매장↔워커 단발 채팅
13. 점주 워커풀 CRM(재고용률·노쇼·별점 정렬) + 어드민 KPI(1시간 매칭률·30분 입금률)

### 4.2 기술 스택

- **백엔드**: Spring Boot 3.5 / Java 17 / MySQL 8 / AWS S3 / 카카오 Local & OAuth API
- **프론트**: Expo Router 6 / React 19 / React Native 0.81 / TypeScript (web + iOS + Android 단일 코드베이스)
- **인프라**: 현재 dev 환경. 운영 전환 시 Railway/AWS + Flyway + EAS Build

### 4.3 보안·컴플라이언스 (현재 + 운영 전 정비 항목)

- **현재**: HTTPS, BasicAuth + bcrypt, AWS IAM 키 관리
- **운영 전**: JWT 전환 / Flyway 마이그레이션 / Toss·오픈뱅킹 PG 실연동 / Apple·Google 스토어 등록

## 5. 비즈니스 모델 (Business Model)

### 5.1 수수료 구조

| 항목 | 요율 | 부담자 |
|------|------|--------|
| 매칭 수수료 | **12%** (시프트 거래액 기준) | 점주 |
| 일용근로 원천징수 | 6.6% (일급 15만원↑) / 0원 (미만) | 워커 (자동 처리) |
| 결제 수수료 | 약 2~3% | 단바 (PG 연동 비용) |

**거래 1건 예시 (시급 ₩12,000 × 5시간 = ₩60,000)**
- 워커 수령액: ₩60,000 (15만원 미만 → 원천징수 0)
- 점주 결제액: ₩60,000 + ₩7,200 (수수료 12%) = ₩67,200
- 단바 매출: ₩7,200 (PG 수수료 약 ₩1,800 차감 시 net ₩5,400)

### 5.2 추가 수익 라인 (Phase 2~3)

- **프리미엄 점주 구독** — 단골 워커 무제한 초대, 우선 매칭 푸시, 통계 PDF 자동 발송
- **광고/위치노출** — 시프트 검색 상단 노출 (Google 검색 광고 모델)
- **검증 워커 풀** — Verified Barista 등급제, POS 사전 교육 모듈 (워커 멤버십)
- **데이터 라이선스** — 카페 상권 분석 데이터 (B2B SaaS 부가)

## 6. 시장 진입 전략 (Go-to-Market)

### 6.1 Wedge — 신촌·홍대 카페 100곳 (3개월)

- **선정 이유**: 대학가 + 카페 밀집 + 알바 수요 빈발 + 워커 풀(20대) 자동 확보
- **영업 방식**: 점주 직접 방문 + 가입 인센티브(첫 3건 수수료 면제) + 카카오 매장 채널 활용
- **목표**: 점주 100곳 활성화 → 월 시프트 1,500건 → MoM 30% 성장

### 6.2 워커 풀 확보

- **초기**: 신촌·홍대·이대 인근 대학교 커뮤니티(에브리타임·캠퍼스픽) 광고 + 친구 초대 ₩3,000 캐시백
- **중기**: 카카오 로그인 1탭 가입 + 기존 알바몬·당근알바 워커 마이그레이션 캠페인
- **장기**: 검증 워커 멤버십(Verified Barista) → 워커 자체 lock-in
